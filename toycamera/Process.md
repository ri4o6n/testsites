# Process - レトロトイカメラWebツール

## 1. コンセプトと方針の決定
- **目的**: Web上で動作する、ランダム性とリアルな質感を兼ね備えた「レトロトイカメラ」風の画像処理ツールの作成。
- **構成**: `testsites` プロジェクトの一部として開発。ViteやReactなどのビルドツールは使わず、既存の構成に合わせたプレーンなVanilla HTML/CSS/JSで実装する。
- **UI/UX思想**: 必須のデザインルール（静かで上質、生活空間に溶け込むミニマルデザイン）を順守し、過剰な装飾を排したUIとする。
- **開発戦略**: いきなり完全なランダム化やWebGLの導入はせず、まずはCanvas 2Dで確実な画像処理パイプライン（MVP）を構築する。
- **拡張性**: 将来的に撮影時アプリへ移植可能なように、画像処理ロジックの純関数化やパラメータの分離を前提とした設計を行う。

## 2. 実装の進捗
- **Phase 1 (MVP) 完了**:
  - `index.html`, `style.css`, `script.js` のファイル群を作成。
  - 画像読み込み・最大解像度（2048px）での自動リサイズとプレビュー表示機能を実装。
  - Canvas 2D APIを用いた基礎的な画像処理パイプライン（微弱なブラー、コントラスト強調、RGBオフセットによる暖色化、彩度アップ、周辺減光、粒子ノイズ添加）を純関数的なアプローチで実装。
  - ルートの `testsites/index.html` へツールへのリンクカードを追加。

## 3. 次のアクション
- ユーザーによるブラウザの動作確認待ち。
- 結果に応じて、Phase 2（トイカメラらしさ強化：白飛び、ランダム色かぶり、光漏れなど）や Phase 3（UIによるパラメータ操作・保存機能など）へ移行する。
