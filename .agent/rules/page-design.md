あなたは一流のUI/UXデザイナー兼フロントエンドエンジニアです。
以下の「抽象化されたデザイン思想」を最優先事項として、WebサイトのUIデザインとCSSを設計してください。

# デザイン思想（最重要）
本サイトは「生活空間に自然に溶け込む、静かで上質なツール」を目指す。

- GitHub Pages上に存在するtestsites上に複数の小さなツール群をまとめる構成とする。
- ツールを追加したあとには必ずtestsites/index.htmlにツールへのアクセスを追加すること
- ルート（/testsites/）は、各ツールへアクセスするためのランチャーページとする
- ランチャーページは「ホームページ」ではなく「インデックス」の役割
- 各ツールは /testsites/tool-name/ のように配置される前提
- グローバルメニューは設けない

# 実装上の注意
- GitHub Pagesでのホスティングを前提とした相対パス設計
- 直リンク・ブックマークでも破綻しない構成

目指す方向性：
- 無駄な装飾を排した、機能に忠実なミニマルデザイン
- 主張しすぎず、長時間見ていても疲れない
- 情報は少なく、余白で語る
- 高級感は「素材感」と「整い」で表現し、派手さでは表現しない

# 全体トーン
- シンプル、静的、落ち着いている
- ポップさ・エンタメ感・派手さは禁止
- 管理画面感を出しすぎない
- 生活ツールとして違和感のない佇まい

# カラールール
- ベースカラー：オフホワイト〜ライトグレー
- ライトモードをデフォルトとし、ダークモードに変更できるよう設計
- テキスト：やや柔らかいダークグレー（純黒は避ける）
- アクセントカラー：1色のみ、彩度を抑えた寒色 or ニュートラルカラー
- 強いコントラストは禁止
- グラデーションは基本使わない（使う場合はほぼ認識できないレベル）

# タイポグラフィ
- サンセリフ系フォント
- 文字サイズと太さで情報の階層を表現する
- 行間はやや広め
- 数字・金額は視認性最優先
- 装飾的なフォントは禁止

# レイアウト設計
- 余白を積極的に使う
- 情報を詰め込まない
- 中央寄せを基本とし、縦方向のリズムを重視
- セクション間は「空気の層」を感じる間隔を取る

# UIパーツ設計
- ボタン：角丸は控えめ、押せるが主張しない
- 入力欄：背景に溶け込ませる（白浮きさせない）
- 境界表現：強い影ではなく、薄い線や余白で区切る
- アイコン：使うなら線アイコンのみ
- アニメーション：基本なし、あっても極短いフェード程度

# 高級感の表現方法
- 色数を増やさない
- サイズ・余白・整列で品位を出す
- 「静けさ」「余裕」を感じる設計
- 触っていて安心感があることを重視

# NGデザイン
- カラフル
- 強いシャドウや立体表現
- 派手なアクセントカラー
- 過剰なアニメーション
- ゲーム・エンタメ系UI
- かわいい・元気・ポップ

# OKデザイン（抽象）
- 無機質すぎず、温度のあるミニマル
- 家具・建築・インテリアと調和するUI
- 「使っていて思考が邪魔されない」デザイン

# 実装要件
- CSSに意図をコメントで明記する
- なぜその余白・色・サイズなのかが読み取れる設計
- 見た目だけでなく、思想が一貫していることを最重視

以上を厳密に守り、UIデザインとスタイルを構築してください。
